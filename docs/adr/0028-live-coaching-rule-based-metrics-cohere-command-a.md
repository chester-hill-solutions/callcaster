# Live coaching via rule-based metrics + Cohere Command A on 30s cadence

Live coaching is **self-service**: the human agent toggles it on per-workspace via `workspace.featureFlags.liveCoaching`, and the system surfaces metrics and optional LLM suggestions — there is no autonomous AI coach orchestrator and no AI voice on the call. Coaching has two layers. **Layer 1 (rule-based, per utterance)** runs in the media-stream service (ADR-0030) on every Deepgram `speech_final` result: rolling WPM over a 30-second window compared against `workspace.coachingConfig.wpmMin`/`wpmMax` (default 120-160), filler-word counter using the configurable `coachingConfig.fillerWords` list (default `['uh','um','like','you know','basically','actually']` — Deepgram's `filler_words=true` tags the first six natively, the rest are matched on transcript text), and pause detector using `utterance_end_ms` gaps and word-timestamp gaps beyond `coachingConfig.pauseThresholdMs` (default 1500ms). These metrics update the UI instantly via pg-realtime SSE (`coaching_metrics` event type) and emit `coaching_event` rows (`type='filler_burst'` when ≥3 fillers in one utterance, `type='pace'` when WPM drifts outside range for 3+ consecutive utterances, `type='pause'` when a long pause is detected). **Layer 2 (LLM, on cadence)** runs in the media-stream service on a configurable `coachingConfig.llmCadenceMs` interval (default 30000ms): the last N transcript segments + current metrics are built into a short prompt and sent to **Cohere Command A** via the `cohere-ai` SDK, which returns `{heading, suggestion}` — a 2-word heading ("Slow down", "Open question", "Ask for the vote") and a one-sentence actionable suggestion. The cue is published as a `coaching_cue` SSE event and inserted as a `coaching_event` row (`type='suggestion'`). The agent sees cues in a CallScreen coaching panel and can acknowledge them (sets `coaching_event.acknowledgedAt`). On stream stop, a `coaching_session` row is written with `wpmAvg`, `fillerCount`, `pauseCount`, `longPauseCount`, `score` (composite: 30% filler frequency, 25% pace adherence, 25% pause discipline, 20% silence ratio — matching the FluentFlow weighting validated in speech-coaching literature), and an LLM-generated `summary`. Cohere Command A is chosen over OpenAI GPT-4o-mini despite `openai@^6.38.0` being a (dead) v1 dep, because v2 is a clean rebuild with curated deps and the user has a Cohere preference — Command A is comparable in quality and cost for short coaching prompts (~$0.15/1M input tokens). Coaching is billable: each cue debits 0.1 credits via the idempotent ledger (ADR-0006), worst case ~120 cues/hour = 12 credits/hour. Compliance disclosure is optional (`coachingConfig.disclosureEnabled`, default false) — when true, the TwiML prepends `<Say>This call may be analyzed by AI for quality purposes.</Say>` before the `<Dial>`/`<Conference>`. Canada is one-party-consent for recording (CONTEXT.md); AI-analysis disclosure is a separate legal question left to the operator.

## Considered Options

- **Autonomous AI coach orchestrator (Phase 2)** — a long-running LLM agent that watches the transcript, decides when to push cues, detects objections, summarizes without a human supervisor. Explicitly deferred: starts with self-service metrics + LLM cadence (Phase 1), add autonomy later. On Node/Bun there's no special agent SDK required — it's just a loop with an LLM client.
- **Cloudflare Agents SDK / Durable Objects** — would introduce Cloudflare Workers infra into a Bun-on-Railway stack with zero Cloudflare footprint. Rejected — an autonomous coach on Bun is just a long-running loop.
- **OpenAI gpt-4o-mini for the LLM** — `openai@^6.38.0` was a dead v1 dep (never imported). v2 is a clean rebuild; using Cohere Command A keeps the dep tree curated and aligns with the Cohere Transcribe batch path (ADR-0029) — one vendor, one SDK (`cohere-ai`), both coaching and transcription.
- **Rules only, no LLM** — cheapest, no hallucination risk, but cues are generic ("Slow down", "You said 'uh' 3 times") not context-aware. Rejected as the sole layer; kept as Layer 1 for instant metrics.
- **Audio whisper / second channel (AI speaks to agent only)** — Twilio conference participant with `muted=true` + `coach=true` flags (ADR-0012 M3). Higher latency, more Twilio wiring, caller might hear. Rejected for Phase 1; the on-screen panel is non-intrusive and ships faster.
- **Post-call report only (no live cues)** — simplest, no real-time pressure, but loses the headline value (live coaching during the call). Rejected as the sole mode; the post-call `coaching_session` summary is included as part of this ADR.

## References

- Reference implementations studied: `Simple-biz/simple-biz-call-coach` (Deepgram → GPT-4o-mini, 3-min warmup + 30s cadence, ~$0.045/call), `Raj-G07/Communication_expert` (Gemini Realtime + Deepgram STT + ElevenLabs TTS, 30s snapshot loop), `bobthearsonist/meeting-coach` (Faster-Whisper + Ollama, WPM 120-160 target, filler tracking), `JithuMon10/FluentFlow` (Web Speech API, 30% filler / 25% stammer / 25% pace / 20% silence scoring — the weighting this ADR adopts), `yaotingchun/VoxLab` (Google Chirp 2, contextual pause analysis distinguishing mid-sentence hesitation from intentional emphasis)
- Deepgram filler words: transcribes `uh`, `um`, `mhmm`, `mm-mm`, `uh-uh`, `uh-huh`, `nuh-uh` with fixed spelling regardless of spoken duration; Nova-3 only; stripped by default, opt-in via `filler_words=true`
- ADR-0006 (billing — coaching cue debit is a Drizzle tx: idempotent ledger insert + `SELECT FOR UPDATE` credits update, key `coaching:${callId}:${cueId}`)
- ADR-0027 (live transcription — this ADR consumes the Deepgram results the transcriber produces)
- ADR-0029 (post-call Cohere batch — the `coaching_session.summary` may use the golden transcript when available)
- ADR-0030 (media-stream service — both coaching layers run there, not on the web server, to isolate CPU)
